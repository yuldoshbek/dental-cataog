# Инструкция для Надыра — Dental Catalog

## Что это
Веб-каталог стоматологического оборудования для менеджеров по продажам.
- **Frontend**: React SPA
- **Backend**: Node.js 22 + Express + SQLite
- **Deploy**: Docker Compose + Nginx

---

## 1. Требования на сервере

```bash
# Проверить версии
docker --version       # 24+
docker-compose --version  # v2.x
node --version         # 22+ (для ручного запуска без Docker)
nginx --version        # если не через Docker
```

Если не установлено:
```bash
# Docker
curl -fsSL https://get.docker.com | sh
usermod -aG docker $USER  # добавить пользователя в группу docker

# Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt-get install -y nodejs
```

## 3. Первоначальный деплой на сервер

```bash
# На сервере — создать директорию
mkdir -p /var/www/dental-catalog
cd /var/www/dental-catalog

# Клонировать репозиторий
git clone https://gitlab.intelnet24.ru/implant-expert/dental-catalog.git .

# Создать директории для данных
mkdir -p data uploads
```

---

## 4. Настроить переменные окружения

```bash
# Скопировать шаблон
cp server/.env.example server/.env

# Отредактировать
nano server/.env
```

**Содержимое `server/.env`:**
```env
NODE_ENV=production
PORT=3001

# ОБЯЗАТЕЛЬНО сменить!
JWT_SECRET=сгенерируй_случайную_строку_длиной_32_символа

# Учётные данные администратора
ADMIN_USERNAME=admin
ADMIN_PASSWORD=сильный_пароль_здесь

# CORS — домен фронтенда (если на том же сервере — оставить пустым)
CORS_ORIGIN=

# WhatsApp менеджера (только цифры, без +)
WHATSAPP_MANAGER_PHONE=79001234567

# Максимальный размер фото (MB)
MAX_FILE_SIZE_MB=10
```

Для `JWT_SECRET` можно использовать:
```bash
openssl rand -hex 32
```

```bash
# Заменить домен в конфиге
sed -i 's/yourdomain.com/ТВОЙ_ДОМЕН/g' nginx.docker.conf

# Или отредактировать вручную
nano nginx.docker.conf
```

## 6. Запустить Docker Compose

```bash
cd /var/www/dental-catalog

# Первый запуск (пересобрать образы)
docker-compose up -d --build

# Проверить статус
docker-compose ps

# Проверить логи API
docker-compose logs -f api

# Health check
curl http://localhost:3001/health
```

## 7. SSL (Let's Encrypt)

```bash
# Установить certbot
apt install certbot python3-certbot-nginx -y

# Получить сертификат (сначала остановить web контейнер)
docker-compose stop web
certbot certonly --standalone -d ТВОЙ_ДОМЕН

# Раскомментировать HTTPS блок в nginx.docker.conf
nano nginx.docker.conf

# Запустить обратно
docker-compose start web

# Автообновление (certbot cron уже настраивается автоматически)
certbot renew --dry-run
```

## 8. Настроить GitLab CI/CD (автодеплой)

### 8.1 Создать SSH-ключ для деплоя

```bash
# На сервере
ssh-keygen -t ed25519 -f ~/.ssh/gitlab-deploy -N ""
cat ~/.ssh/gitlab-deploy.pub >> ~/.ssh/authorized_keys
cat ~/.ssh/gitlab-deploy  # скопировать приватный ключ
```

### 8.2 Добавить переменные в GitLab

Перейти в GitLab → проект → **Settings → CI/CD → Variables → Add variable**:

| Переменная | Значение | Protected | Masked |
|-----------|---------|-----------|--------|
| `SSH_PRIVATE_KEY` | содержимое `~/.ssh/gitlab-deploy` | ✅ | ✅ |
| `SSH_USER` | имя пользователя на сервере | ✅ | - |
| `SSH_HOST` | IP или домен сервера | ✅ | - |
| `APP_DIR` | `/var/www/dental-catalog` | ✅ | - |

### 8.3 Установить GitLab Runner (на сервере)

```bash
# Установить runner
curl -L "https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.deb.sh" | bash
apt-get install gitlab-runner

# Зарегистрировать runner
gitlab-runner register
# URL: https://gitlab.intelnet24.ru/
# Token: взять из Settings → CI/CD → Runners
# Executor: shell
# Tag: dental-server
```

После этого — при каждом `git push origin main` GitLab автоматически:
1. Собирает frontend
2. Копирует dist/ на сервер
3. Пересобирает Docker образ API
4. Перезапускает контейнеры
5. Проверяет health check

## 9. Полезные команды

```bash
# Просмотр логов
docker-compose logs -f api
docker-compose logs -f web

# Перезапуск
docker-compose restart api

# Полная остановка и запуск
docker-compose down
docker-compose up -d

# Пересборка без кеша
docker-compose build --no-cache api
docker-compose up -d

# Бэкап базы данных
cp /var/www/dental-catalog/data/dental.db /backup/dental-$(date +%Y%m%d).db

# Посмотреть использование ресурсов
docker stats
```
